# Симуляция
Симуляция 2-D мира, где есть хищники, травоядные, съедобные растения и прочие статические объекты.

## Описание проекта
Симуляция — это бесконечный цикл, где хищники охотятся за травоядными и грызунами, а те, в свою очередь, заняты поиском съедобных растений. Хищники выбирают жертву под стать своему размеру: Кот ищет Мышей, Волк может съесть Кролика или Овцу, Лев нападает на Овец, Зебр и Буйволов, а Тиранозавру интересны только самые крупные травоядные — Зебра и Буйвол.

Всего представлено 13 видов сущностей. 

- Хищники: Кот, Лев, Волк, Тиранозавр.
- Травоядные: Буйвол, Зебра, Овца.
- Грызуны: Кролик, Мышь.
- Съедобные растения: Трава, Зерно.
- Статические объекты: Дерево, Скала.

Если какого-то вида Сущностей стало критически мало (< 30% от начального значения), происходит их восполнение. Трава и Зерно прорастают в произвольных местах в количестве не более первоначального. Новые животные заходят с края карты (появляются в пустых ячейках по периметру карты). В случае полного исчезновения за один ход всех Сущностей одного вида, идёт их полное восполнение до начального уровня в произвольных местах карты. 

## Взаимодействие Существ
Каждое Существо создаётся с разными параметрами, в пределах, доступных для своего вида:

- Травоядные и Грызуны — здоровье, скорость; 
- Хищники — здоровье, скорость, сила атаки;
- Съедобные растения — пищевая ценность.

Например, Волк, создаётся с такими значениями: Здоровье — 80…100, Скорость (максимальное кол-во ячеек для перемещения за один ход) — 4…5, Атака — 30…70. В процессе охоты, он может улучшить своё здоровье до максимально возможного для его вида — 200.

При нападении Хищник забирает у добычи здоровье на величину своей Атаки. Если при этом здоровье добычи уменьшается до 0, то она исчезает с Карты и Хищник становится на её место. Если добыча остаётся в живых, то Хищник становится в ячейке рядом с ней.

При поедании Съедобных растений, здоровье увеличивается на величину их пищевой ценности.

Если еда не была найдена, то Существо перемещается на произвольную ячейку в соответствии со своей скоростью, его здоровье при этом уменьшается. При достижении нулевого уровня здоровья, Существо умирает и удаляется с Карты мира.

## Управление Симуляцией
Управление происходит через Основное меню, предлагающее выбор одной из трёх конфигураций Карты Мира: 

- Маленькая Карта – размер 15 х 15, только Волки и Кролики. В консоль выводится информация о каждом ходе: ID Сущности, его координаты и состояние (уровень здоровья, скорость, сила атаки для хищников), найдена ли цель, координаты цели и путь к ней, состояние Сущности после движения. 
- Базовая Карта — размер карты и процент заполнения ячеек карты задаётся пользователем (с учётом встроенных ограничений). Создаются Волки, Кролики, Коты и Мыши. Их соотношение между собой задавать нельзя. Логирование в консоль отключено. 
- Произвольная Карта — размер карты и количество каждого вида Сущностей (13 видов) задаётся пользователем (с учётом встроенных ограничений). Логирование в консоль отключено.

После запуска пользователь может включать/выключать паузу (s / s + Enter) или полностью прервать выполнение (q) Симуляции.

## Основная техническая цель проекта
Демонстрация принципов дизайна архитектуры приложения с помощью ООП.

## Некоторые технические детали
- Java 11
- Maven 3.9.6
- Зависимости: org.reflections  и org.slf4j

У Reflections использован метод .getSubTypesOf() для получения списка наследников класса.

Использованы паттерны проектирования: Фабрика, Синглтон, Шаблонный метод.

## Установка и запуск
- Клонируйте репозиторий:
git clone https://github.com/dotogo/simulation.git
- Откройте проект в IntelliJ IDEA
- Постройте проект с помощью Maven:
mvn clean install

После установки запустите Main, чтобы начать Симуляцию.
